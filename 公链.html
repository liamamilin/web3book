The key of web3

Chapter 3 公链

为什么是通用计算服务？

在深入探讨之前，我们先回答一个关键问题：为什么称其为“通用计算服务”？

这是因为 Web 3.0 的计算层具有开放性，任何人都可以使用它。不同于目前绝大部分网站运行在专属服务器上的模式，Web 3.0 将计算层独立出来，类似于我们使用的公共基础设施，如 5G 网络、自来水系统、电网等。

为什么要独立计算层？
Web 3.0 强调计算层独立的主要目的是通过分层架构防止数据垄断，让数据以公共账本的形式公开共享。当然，数据公开并不是终点，最终目标是让数据的所有权回归数据创造者。但要实现这一目标，第一步是解除对数据的垄断。

以太坊是这种通用计算服务的典型代表。作为一种公链，它的计算网络开放而去中心化，任何人都可以自由加入或退出，无需受到任何限制。这种特性使得以太坊成为去中心化应用（DApps）的重要基础设施。接下来，我们以以太坊为起点，探讨公链如何提供通用计算服务。

3.0.1 从比特币到以太坊

要理解以太坊，首先需要从比特币说起。比特币通过去中心化网络首次实现了确定性的计算服务。尽管被称为“数字黄金”，从技术角度看，比特币其实是一个“状态转换系统”：它以比特币交易信息为输入，通过状态转换函数输出更新后的比特币账本。

这个过程与银行的账务处理系统有一定相似性。比特币在不同时刻的账本状态相当于银行的资产负债表，而“状态转换函数”则相当于银行的账务处理逻辑。

然而，比特币的局限性也显而易见： 1. 扩展性不足：比特币的脚本语言只能支持简单的逻辑操作，缺乏复杂运算能力，无法实现图灵完备。 2. 功能受限：比特币仅支持一次性合约，无法处理多阶段期权合约或去中心化交易等复杂应用。

以太坊正是为了解决这些问题而设计的。它建立在比特币的基础之上，提供了更强大的计算性能、更简便的开发环境，以及与其他应用共享区块链安全性的能力。

3.0.2 以太坊为何被称为“世界通用账本”？

以太坊内置了图灵完备的编程语言，使任何人都可以基于其平台编写智能合约和开发去中心化应用（DApps）。用户还能够自由定义所有权规则、交易形式以及状态转换逻辑。

与比特币不同，以太坊采用了账户模型。账户不仅记录资产余额，还支持与合约的交互。这一设计显著提升了智能合约的效率和灵活性，同时为以太坊的扩展性奠定了基础。

应用场景：
1. 复杂合约逻辑：比特币支持简单的多签名支付，而以太坊可以实现更精细的控制逻辑。例如，五个账户中需三个签名才能提取80%的资金，两人签名则每天最多提取10%的资金。 2. 去中心化金融（DeFi）：以太坊为代币发行、储蓄借贷、金融衍生品等 DeFi 应用提供支持，成为 DeFi 生态的核心基础设施。

因此，以太坊不仅是去中心化应用的“操作系统”，还因其严格同步的账户状态被称为“世界通用账本”。

3.0.3 IPFS：为存储层提供通用解决方案

如果以太坊提供了 Web 3.0 的通用计算层，那么 IPFS则代表了通用存储层。

IPFS 的核心机制：
IPFS 将文件分块（默认大小为 256KB），为每个文件生成唯一的哈希值。文件的哈希值与内容绑定，形成完整的索引结构，存储在分布式网络中。当用户发出检索请求时，节点通过索引找到对应文件。

优势： 1. 分布式存储：文件存储不依赖特定节点或中心化服务器，避免因服务器故障或被攻击导致数据不可用。 2. 数据所有权回归用户：用户根据内容检索文件，打破了传统互联网“存储即拥有”的模式，赋予用户更多的数据控制权。 3. 经济激励：IPFS 整合分散的存储空间，为个人和企业创造经济收益的机会。

实际案例： 1. NFT 存储：许多 NFT 项目（如 CryptoPunks 和 Bored Ape Yacht Club）选择 IPFS 作为图片存储方案，避免中心化存储的篡改或丢失风险。 2. 以太坊官网迁移：2022 年，以太坊官网正式部署到 IPFS，并通过 ENS 域名实现访问，展现了 Web 3.0 协议间的组合性。

3.1.1 Internet Computer：专为 Web 3.0 应用而生的公链

Internet Computer（IC）进一步扩展了去中心化应用的边界。它旨在为 Web 3.0 提供全栈式开发和运维服务，使去中心化应用的覆盖范围更加广泛。

IC 的技术特点： 1. 全栈式支持：从前端到后台，IC 提供了一套完整的去中心化开发框架，减少对中心化节点的依赖。 2. 更高效的智能合约：克服传统区块链在速度、存储成本和计算效率上的局限，提供接近中心化网络的运行速度。 3. 互操作性：IC 支持与比特币、以太坊等网络的原子级互操作性，实现了跨链应用的高效协作。

尽管 IC 仍处于早期发展阶段，其生态系统尚待完善，但其技术设计展现了去中心化应用的巨大潜力。

3.1.2 以太坊仍是领导者

虽然其他公链不断创新，但以太坊仍然占据 Web 3.0 的核心地位。自 2018 年起，以太坊便启动了升级计划，包括从 PoW（工作量证明）向 PoS（权益证明）的转换，以及通过分片技术实现并行计算。这些技术突破有望显著提升以太坊的性能和扩展能力。

此外，以太坊还催生了 Layer-2 解决方案（如 ZK Rollup 和 Optimistic Rollup），通过将部分计算移至链下，提升链上效率。这些技术虽未完全成熟，但为以太坊生态注入了新的活力和发展潜力。

3.1.3 总结

通用计算服务是 Web 3.0 的核心，它以开放、去中心化的架构重新定义了互联网的技术基础。以太坊和 IPFS 分别为计算层和存储层提供了强大的支持，而 Internet Computer 则为去中心化应用探索了更多可能性。

随着技术的不断迭代，Web 3.0 不仅推动了去中心化生态的崛起，也为用户和开发者创造了前所未有的机会。无论是通用计算服务，还是分布式存储，这场变革的核心在于打破垄断、还原数据所有权，同时为未来的互联网构建更加公平和高效的基础设施。
